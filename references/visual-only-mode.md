# Visual-only mode

## Objective

Upgrade imagery and visual language without changing the deck's structure or audience-facing meaning.

## Preservation baseline

Before editing, record:

- source slide count and order;
- visible title and body copy by slide;
- numbers, units, dates, sources, and quotations;
- native charts, tables, and image relationships;
- internal notes and placeholder instructions that must not remain audience-visible.

Create a one-to-one plan. Each output page inherits the same communication job as its source page.

## Allowed changes

- palette and background treatment;
- typography family, weight, hierarchy, line spacing, and alignment;
- spacing, margins, crop, image scale, and decorative accents;
- replacement of image placeholders with relevant final imagery;
- conversion of a low-density source page into an editorial full-bleed composition;
- visual hiding of production notes, TODOs, generation prompts, and comments;
- line breaks needed for fit, without altering wording.

## Forbidden changes

- changing page order;
- adding or removing audience-facing sections;
- merging or splitting pages;
- shortening or rewriting business copy without explicit authorization;
- replacing a specific project image with unrelated stock imagery when the source image is usable;
- deleting numbers, sources, limitations, or qualifications;
- changing a slide's claim to fit a preferred template.

## Image replacement procedure

1. Reuse original project imagery when it communicates the intended case.
2. Improve crop, scale, color treatment, pairing, and caption hierarchy.
3. Use supplied or licensed imagery for generic atmosphere only.
4. Generate missing case visuals only after confirming the source deck lacks them.
5. Keep every generated image free of presentation text and brand marks.

## Internal-note treatment

Classify candidate text as:

- audience copy;
- speaker note;
- production instruction;
- image prompt;
- unresolved placeholder;
- uncertain.

Hide or replace production instructions and image prompts. Never silently remove uncertain text; record it for review.

Common internal-note patterns include requests to insert a screenshot, generate an image, replace a case, show a recording, ask a colleague, or decide content later.

## QA interpretation

Slide-count and visible-copy preservation are hard gates. Rhythm, dark-page ratio, and template-diversity checks are advisory when satisfying them would require structural changes.
